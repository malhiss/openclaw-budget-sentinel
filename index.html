Why this matters

        
The model is imperfect (80%). It does not matter, because the deterministic gate owns the verdict: an imperfect advisory can only ever route to more human review, never to unauthorized spend. The safety floor is 0% unsafe by construction, independent of which model you run. A bigger model (Qwen3.6-35B-A3B or Kimi K2.6) raises the exact-match rate; it cannot lower the safety floor, because the floor is not the model's job. That is the thesis: govern the model, do not trust it.

      

    

  


  
  

    

      
How it works

      
A governed gate on every action, self-hosted by design.

      
Agents that can spend money are the control risk that stalls enterprise adoption. The gate makes that risk governable: the model can never approve, act, or exceed a cap, and any failure routes to a human, never to an approval.

The report

      
The whole assessment, in brief.

    

    

      
Selected model
Kimi K2.6 (open-weight, the cloud capability ceiling: evaluated from published benchmarks and pricing) and Qwen (open-weight, self-hosted: Qwen3-4B ran every number on this page). Kimi is a paid API and its 1T weights are heavy to host, so I ran the free, self-hostable Qwen and cite Kimi.

      
In plain words
Open-weight models are AI models you can run on your own hardware instead of a paid cloud API. That keeps client spend data in-house and makes cost predictable, which is exactly what OpenClaw's governed swarm needs for regulated clients.

      
Use case
Govern the spend of OpenClaw's 52-agent swarm: routine top-ups auto-clear, over-cap or suspicious items go to a human, fraud is blocked, and every decision is written to an audit ledger. The 80/20 model, made enforceable.

      
Departments
Finance (hard caps + audit trail) · Engineering (a reusable governed-agent pattern) · IT/Infra (self-hosted, no data egress) · Product (a shippable OpenClaw module for pitches) · Strategy (no vendor lock-in, data-residency ready) · Operations (people touch only the ~20% of exceptions).

    


    

      
Risks & limitations

      

        
Privacy: self-hosted, so spend data never leaves the box.

        
Model accuracy: 80% on a small 4B; every error was safe-direction (to a human, never auto-approved). A bigger model raises it.

        
Prompt injection: field text is treated as untrusted data and flagged to a human; in production the amount comes from the authoritative PO record, not the model.

        
Hallucination: the prompt forbids inventing figures; a fabricated high cost still hits a cap and routes to a human.

        
Hardware / integration: size the model to the tier; a clean classify() boundary swaps mock, local, or cloud by one env var.

        
Reliability: fail-closed by design: any model or parse error routes to a human, never to an approval.

Final recommendation

        
Test it — a controlled internal pilot.

        
    
          
  • Pilot one OpenClaw branch on self-hosted Qwen3.6-35B-A3B, capped budget, humans clearing every hold for two weeks.
    • 
          
  • Set ship criteria: unsafe rate stays 0, exact-match above 90% on a larger, adversarial set.
    • 
          
  • Operationalise the 20%: an approval SLA and a monthly ledger-verification check.
    • 
          
  • Expand branch by branch, only after the ledger proves the safety and cost case.
